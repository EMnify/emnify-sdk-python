Examples#

Get First Devices Online#

In order to get your devices online, you gonna need to perform a set of steps below. This example demonstrates complex operation across multiple SDK Concepts.

  • Device Policies that configure which services and where are gonna be available.

  • You can learn more about APN configuration via SMS in this article.

  • SIM Registration

  • Device Creation

  • SIM Assignment

  • Error Handling

  • Retrieve Device Details

  • Device Activation/Deactivation

mass_sim_activation.py#
 1from emnify import EMnify
 2from emnify import constants
 3from emnify.errors import EMnifyBaseException
 4
 5# === Example: Getting First Devices Online ===
 6
 7# To operate EMnify SDK you need to generate application token.
 8# That is how: https://www.emnify.com/developer-blog/how-to-use-an-application-token-for-api-authentication
 9token = input('token: ')
10# The client is authorized to perform operations by your name;
11emnify_client = EMnify(token)
12
13# There are two things you need to obtain before you get you device online,
14# one is a device and other is a SIM card.
15# For this example we assume you have a batch of SIM cards you willing to
16# use in your devices.
17sim_batch_BIC2 = input('BIC2: ')
18
19# EMnify allows you to control coverage and services you need.
20# Lookup Concepts to learn more.
21# https://portal.emnify.com/device-policies you can find IDs there.
22service_profile_id = input('Service Profile ID: ')
23tariff_profile_id = input('Tariff Profile ID: ')
24# In order to create devices for SIMs afterwards we need service and coverage profiles.
25service_profile = emnify_client.devices.service_profile_model(id=int(service_profile_id))
26tariff_profile = emnify_client.devices.tariff_profile_model(id=int(tariff_profile_id))
27
28try:
29    # Having a batch of SIM cards you need to add them to your account;
30    # This method also supports single sim registration via BIC1,
31    # so you can use your FREE Evaluation SIM cards either;
32    issued_sims = emnify_client.sim.register_sim(bic=sim_batch_BIC2)
33    # After it's added - all sim`s are registred with "issued" status;
34except EMnifyBaseException as e:
35    # It can happen that there is some error appear during SIM registration;
36    # Use EMnifyBaseException for general exceptions or inherited classes for specific one`s;
37    raise AssertionError(f"error during sim batch BIC2 activation{e}")
38
39# We define status of device to be applied during creation.
40device_status = emnify_client.devices.status_model(
41    **constants.DeviceStatuses.ENABLED_DICT.value
42)
43
44
45for sim in issued_sims:
46    # SIM cards do not provide connectivity by just registering them,
47    # So in order to get devices you put SIM cards in you also need to
48    # create a new device with the SIM assigned.
49
50    # For device creation we need to specify parameters of the device:
51    device_name = f"Device({sim.iccid})"
52    device_model = emnify_client.devices.device_create_model(
53        tariff_profile=tariff_profile,
54        status=device_status,
55        service_profile=service_profile,
56        sim=sim,
57        name=device_name
58    )
59    # See API Reference to learn other parameters the device can have.
60
61    # And there is how we create a device we wanted.
62    device_id = emnify_client.devices.create_device(device=device_model)
63
64    # After creation we may retrieve full device information.
65    # Which you can store in your local inventory for future needs.
66    device = emnify_client.devices.retrieve_device(device_id=device_id)
67
68    # As it mentioned above by default connectivity is disabled so you're not getting billed.
69    # The following command will enable connectivity for your device with a SIM card installed.
70    emnify_client.devices.change_status(enable=True, device=device)
71    # At this point EMnify is ready to provide you connectivity services.
72    # So we can already send and receive SMS(if this enabled in the Service Profile assigned).
73
74    # After making sure device is created and enabled
75    # it is time to perform configuration on a device side.
76
77    # Correct APN configuration of the device is required to access internet.
78    # EMnify APN is(no spaces, just two characters): em
79    # For example purposes we will send a special configuration SMS command.
80    # It may vary by the device manufacturer, see our documentation to learn
81    # if this method suites your devices.
82    ACTIVATION_CODE = 'AT+CGDCONT=1,"IP","em",,'
83    SENDER = "city_scooters_admin"
84
85    activation_sms = emnify_client.devices.sms_create_model(
86        payload=ACTIVATION_CODE,
87        source_adress=SENDER
88    )
89
90    # And now we send the configuration SMS to our device.
91    emnify_client.devices.send_sms(device=device, sms=activation_sms)
92
93    # CONGRATULATIONS! You get your device online!
94    # Now you can check your device internet access.

device_lifecycle_management.py#
 1#  === Create and activate a device ===
 2
 3
 4unassigned_sims = [i for i in emnify.sim.get_sim_list(without_device=True)]
 5#  If no unassigned_sims - register new one by batch code
 6if not unassigned_sims:
 7    registered_sim = emnify.sim.register_sim(bic='sample_bic_code')  # Returns as list
 8    sim = emnify.sim.retrieve_sim(registered_sim[0].id)
 9else:
10    sim = unassigned_sims[0]  # Taking the first one unassigned sim
11
12# Defining of new device parameters
13# All required models can be retrieved through manager`s properties
14service_profile = emnify.devices.service_profile_model(id=1)
15tariff_profile = emnify.devices.tariff_profile_model(id=1)
16device_status = emnify.devices.status_model(id=0)
17name = 'new_device'
18device_model = emnify.devices.device_create_model(
19    tariff_profile=tariff_profile,
20    status=device_status,
21    service_profile=service_profile,
22    sim=sim,
23    name=name
24)
25
26# After creating model sdk returns id of device
27device_id = emnify.devices.create_device(device=device_model)
28# Then we can retrieve all device details
29device = emnify.devices.retrieve_device(device_id=device_id)
30# Activate device
31emnify.devices.change_status(device=device, enable=True)
32
33# Retrieving updated device details
34device = emnify.devices.retrieve_device(device_id=device_id)
35device_status = device.status.description  # Will be 'Enabled'
36sim_status = device.sim.status.description  # Will be 'Activated'

Configure a device#

If you want to avoid using a specific operator to avoid unintentional spending on the device, you can add it to the blacklist of the device.

device_lifecycle_management.py#

 4device = emnify.devices.retrieve_device(device_id=device_id)

 6tags = 'arduino, meter, temp'  # Sample tags
 7name = 'new name'  # Sample name

 9# Adjust device config
10update_device_fields = emnify.devices.device_update_model(name='new name', ip_address='192.0.0.0')
11emnify.devices.update_device(device_id=device.id, device=update_device_fields)
12
13#  Getting details of updated device
14updated_device = emnify.devices.retrieve_device(device_id=device_id)

Filtering and Sorting#

filtering_and_sorting.py#
 1
 2# === Using a Filtering for List calls  ===
 3from emnify import EMnify
 4from emnify import constants as emnify_constants
 5
 6emnify_client = EMnify(app_token='your_application_token')
 7
 8# Some methods that return multiple objects allow sorting and filtering.
 9# This allows us to optimize processing time, since using filtering we can immediately get the necessary objects with
10# the necessary qualities, and sorting allows us to set the order in which objects are displayed.
11# Instead of sending several requests in search of the required object, we can reduce their number to one.
12
13# For example - we want to find all SIM`s with specific customer organisation, first one for instance.
14# We need to initiate the model for filtering by specifying the necessary parameters in it as arguments.
15sim_filter = emnify_client.sim.get_sim_filter_model(customer_org=1)
16# To get the filtering model for filling, you need to get it as a property of a manager
17# for SIM cards, this would be get_sim_filter_model, for Devices it would be a get_device_filter_model
18
19# After initializing the model object, it must be passed as an argument that makes a request to get a list of objects.
20sims = emnify_client.sim.get_sim_list(filter_model=sim_filter)
21# sims now contains the objects we need with client organization 1
22
23# We can pass several parameters for filtering for a more detailed search for the necessary objects.
24sim_filter = emnify_client.sim.get_sim_filter_model(
25    customer_org=1,
26    status=emnify_constants.SimStatusesID.ACTIVATED_ID.value,
27    production_date='2019-01-25'
28)
29
30# The request to get a list of SIM cards also has a separate filter,
31# which is passed as an argument to the filtering function - without a device.
32
33sims_without_assigned_device = emnify_client.sim.get_sim_list(without_device=True)
34
35
36# === Using a Sorting for List calls  ===
37# Just like filtering, sorting allows us to reduce processing time by ordering objects in the server.
38# Thus, it is easier for you to group objects according to a certain attribute by specifying it in sorting.
39
40# For example - we want to get all devices sorted by last updated date
41# All sorting is done by using enums
42sort_parameter = emnify_client.devices.get_device_sort_enum.LAST_UPDATED.value
43
44# After choosing a parameter for filtering, we need to pass it as an argument sort_enum
45sorted_devices = emnify_client.devices.get_devices_list(
46    sort_enum=sort_parameter
47)
48
49# Now we got a list of devices, at the beginning of which are the recently updated devices
50for latest_device in sorted_devices:
51    ...
